## 链表
### 定义
-
  #+BEGIN_QUOTE
  链表由一个个数据节点组成，它是一个递归结构，要么它是空的，要么它存在一个指向另外一个数据节点的饮用。
  #+END_QUOTE
-
- 链表的优点是移动和删除数据元素快，只要把相关的数据元素重新链接起来，但缺点是占用空间大，查找需要遍历。
-
## 数组
### 基本数组 array
- 数组是一种基本类型，把相同数据类型的元素按一定顺序排列的集合。
- 它的作用只有一种：存放数据。让你很快就能找到存的数据。如果你不去额外改进它，它就只是存放数据而已，它不会将一个数据节点和另外一个数据节点关联起来。
-
- 它的下标会转换成**虚拟内存地址**，然后操作系统会自动帮我们寻址，这个寻址过程是特别快的，所以往数组的某个下标存取一个值的时间复杂度都为 O(1)
-
- 优点：占用空间小，查询快，直接使用索引就可以获取数据元素，缺点是移动和删除数据元素要大量移动空间。
-
### 可变长数组 slice
- 因为数组大小是固定的，当数据元素特别多时，固定的数组无法储存这么多的值，所以可变长数组出现了。Golang中，可变长数组被内置在语言里面：切片slice。
-
#### slice 结构体
- ``` 
  type slice struct {
	- array unsage.Pointer
	- len int
	- cap int
- }
  ```
-
  1. 指向底层数组的指针（Golang没有操作原始内存的指针，所以unsafe包提供相关的对内存指针的操作）
-
  2. 切片的真正长度，也就是实际元素占用的大小。
-
  3. 切片容量，底层固定数组的长度。
-
- 切片内部维护了一个固定大小的数组，当append新元素时，固定大小的数组不够时会自动扩容。因此可以看到，Golang的切片无法原地append，每次添加完元素时会返回新的引用地址，需要把它赋给之前的变量。并且，当容量不够时，会自动倍数递增扩容（Golang在切片长度大于 1024后，会以接近于1.25倍进行容量扩容）这个1.25倍，据说是参考了实际经验。
-
- tips： slice在容量够时，append的时间复杂度是O(1)，容量不够时，它会创建一个固定大小2*len的新数组来替换老数组，耗时主要在移动老数组的数据到新数组里，O(n)。

## 树
### 平衡二叉树
	- #### 定义
	-
	  #+BEGIN_QUOTE
	  平衡二叉树是基于二分法的策略提升数据查找速度的二叉树
	  #+END_QUOTE
	- #### 数据组装规则
		-
		  1. 非叶子节点只能允许最多两个子节点存在
		-
		  2. 每一个非叶子节点数据分布规则：左子节点值 < 父节点值，右子节点值 > 父节点值。这里的值是基于自己的算法规则而定的，比如hash值）
		- ![](https://ser4wang.oss-cn-beijing.aliyuncs.com/20210808161557.png)
-
	-
	- 平衡的实现关键是其层级结构，因为平衡二叉树的查询性能和树高成反比，通过一种算法机制（Treap、红黑树）来实现节点数据结构的平衡以避免影响查询效率。
	- ![](https://ser4wang.oss-cn-beijing.aliyuncs.com/20210808162820.png)
	- 平衡二叉树的查找时间复杂度同二分查找，O(logN);
	-
	- #### 总结
		-
		  1. 非叶子节点最多拥有两个子节点
		-
		  2. 非叶子节点大于左子节点，小于右子节点
		-
		  3. 树左右两边的层级数相差不会大于1 （防止树高度过高。树越高和查询速度呈反比）
		-
		  4. 没有值相等重复的节点
-
### B树 Balance Tree
	- #### 定义
	-
	  #+BEGIN_QUOTE
	  B树和二叉树稍有不同，是一种**多路查找平衡树**
	  #+END_QUOTE
	- #### 规则
		- 排序方式：所有节点关键字是按递增次序排列，并遵循**左小右大原则**
		- 子节点数：非叶节点的子节点数 满足 (1, M] ，且M >= 2，空树除外（注：M阶代表一个树节点最多有多少个查找路径，M=M路，当M=2则是二叉树，M=3则是三叉树）
		- 关键字数：枝节点的关键字数量 满足 [ceil(m/2), M-1]
		- 所有叶子节点均在同一层
	- ![](https://ser4wang.oss-cn-beijing.aliyuncs.com/20210808164054.png)
	- #### 特点
	- B树相对于平衡二叉树的不同是，每个节点包含的关键字增多了，特别是在B树应用到数据库中的时候，数据库充分利用了磁盘块的原理（磁盘数据存储是采用块的形式存储的）把节点大小限制和充分使用在磁盘块大小范围；把树的节点关键字增多后，树的层级比原来的二叉树少了，减少数据查找的次数和复杂度
